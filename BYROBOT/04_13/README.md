# 📅 2026-04-13 실습: 군집 비행(Swarm Flight) 및 양방향 통신

이번 실습에서는 다수 기체의 통신 상태를 실시간으로 확인하며 제어하는 **군집 비행 시스템**을 구현하고, 통신 신뢰도를 높이기 위한 **ACK(수신 확인) 로직**을 중점적으로 다루었습니다.

## 📂 실습 파일 구성
- `triangle_scan_mission.py`: Yaw 스캔 및 삼각형 경로 순찰
- `diamond_patrol_mission.py`: 대각선 이동 및 코너별 방향 전환
- `altitude_stair_mission.py`: 계단식 고도 변화 및 안정성 테스트
- `swarm_simultaneous_control.py`: 다수 기체 동시 이륙/비행/착륙 제어
- `swarm_sequential_control.py`: 다수 기체 순차적 이륙/비행/착륙 제어
- `dual_drone_led_sequence.py`: ACK 로직을 활용한 2대 드론 LED 순차 변경 테스트

## 🛰️ 핵심 기술: ACK 기반 양방향 통신

단순히 명령을 송신하는 것을 넘어, 드론이 명령을 잘 받았는지 확인하는 검증 로직을 도입했습니다.

1. **상태 추적 (`ack_status`)**
   - 딕셔너리 구조를 사용하여 어떤 포트(기체)가 응답했는지 개별적으로 추적합니다.
2. **이벤트 핸들러 (`event_ack_handler`)**
   - 기체로부터 ACK 응답이 오면 `is_received` 플래그를 `True`로 변경하여 수신 여부를 확인합니다.
3. **재전송 메커니즘 (`send_to_drone_until_ack`)**
   - 응답이 올 때까지 명령을 반복 전송(Timeout 2.0초)하여 통신 성공률을 극대화합니다.

## 🛠️ 트러블슈팅 (Troubleshooting)

실습 중 발생한 하드웨어 및 소프트웨어 이슈와 해결 과정을 정리했습니다.

### ✅ 해결된 상황
| 현상 | 원인 분석 | 해결 방법 |
| :--- | :--- | :--- |
| **3대 이상 연결 시 통신 단절** | 노트북 USB 포트의 전력 공급 부족 | **유전원 USB 허브**에 전원 어댑터 연결 |
| **종료 시 마지막 기체 미작동** | 명령 전달 전 메인 스레드 종료 | `close()` 호출 전후에 **`sleep(2)` 추가**로 시간 확보 |

### ⚠️ 추가 분석 필요 상황
- **명령 로그는 정상이나 물리적 미작동:** 통신 간섭 또는 일시적 전압 저하로 추정되어 추가 분석이 필요합니다.
- **특정 기체의 일부 명령 무시:** 하드웨어 고유 결함 또는 특정 패킷의 로스 가능성을 염두에 두고 있습니다.
